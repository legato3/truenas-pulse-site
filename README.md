# truenas-pulse-site

Promotional site for [TrueNAS Pulse](https://apps.apple.com/app/id6759870893) — the third-party iOS client for TrueNAS SCALE.

Live: **https://pulse.phobos-cc.be**

## Stack

- [Astro 5](https://astro.build) (static, zero JS by default)
- [Tailwind CSS v4](https://tailwindcss.com) — native Vite plugin, `@theme` block holds all brand tokens
- [MDX](https://mdxjs.com) for legal pages (Privacy, Terms, EULA)
- Custom 30-line IntersectionObserver + WAAPI for scroll reveals (no JS animation library)
- [sharp](https://sharp.pixelplumbing.com) for build-time icon and OG image generation
- Bun as the package manager and runtime (swappable for Node 20+ if needed)

## Local development

```bash
bun install
bun run dev          # http://localhost:4321 with hot reload
bun run build        # static build into ./dist
bun run preview      # serve ./dist for a local production preview
```

To regenerate the favicon set, OG image, and webmanifest from the iOS app's
master icon:

```bash
bun run scripts/generate-assets.mjs
```

The script looks for `../TrueNAS-Pulse/assets/icon/TrueNAS_Pulse_AppIcon_Master_UNMASKED_1024.png`
first, then `./public/icon-master.png`.

- `../TrueNAS-Pulse/docs/MONETIZATION.md` — tier copy, pricing rationale
- `../TrueNAS-Pulse/docs/PRO_FEATURES.md` — canonical 26-feature registry
- `../TrueNAS-Pulse/TrueNAS Pulse/Design/Design.swift` — color tokens

When the iOS docs change, update `src/data/features.ts` and any affected
component copy. The brand voice is **calm, deliberate, infrastructure-grade —
never playful**.

## Deployment

`.gitea/workflows/deploy.yml` runs on every push to `main`:

1. Self-hosted runner clones the repo
2. Installs Bun (cached after first run)
3. `bun install --frozen-lockfile`
4. `bun run build`
5. Rsync `dist/` to LXC 132 `prox-web` at `/var/www/pulse-site/`
6. `chown -R caddy:caddy` + `systemctl restart caddy`
7. End-to-end smoke probes 5 pages — fail closed on any non-200

Secrets used:

- `DEPLOY_SSH_KEY` — Ed25519 private key, public side in LXC 132
  `/root/.ssh/authorized_keys`
- `DEPLOY_HOST` — `192.168.0.93`

The Cloudflare tunnel `phobos-home` routes `pulse.phobos-cc.be` →
`http://192.168.0.93:80` (LXC 132 → Caddy 2 serving the static build).
